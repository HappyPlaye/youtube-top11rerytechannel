# YouTube Top Tracker

Следит за твоими любимыми каналами и показывает, какие видео набирают больше всего просмотров за **24 часа / неделю / месяц**. Работает бесплатно и круглосуточно на GitHub — твой компьютер не нужен.

Как это устроено: раз в час GitHub Actions запускает сборщик, который снимает счётчики просмотров всех свежих видео с твоих каналов и сохраняет их. Страница показывает разницу между снимками — то есть реальный прирост за период.

---

## Установка (один раз, ~15 минут)

### Шаг 1. Аккаунт GitHub

Если нет — зарегистрируйся на [github.com](https://github.com) (бесплатно).

### Шаг 2. API-ключ YouTube (бесплатно, карта не нужна)

1. Зайди на [console.cloud.google.com](https://console.cloud.google.com) под своим Google-аккаунтом.
2. Вверху нажми выбор проекта → **New Project** → любое имя → **Create**.
3. Меню ☰ → **APIs & Services** → **Library** → найди **YouTube Data API v3** → **Enable**.
4. **APIs & Services** → **Credentials** → **+ Create Credentials** → **API key**.
5. Скопируй ключ (строка вида `AIzaSy...`) — он понадобится на шаге 4.

Лимит бесплатного ключа — 10 000 юнитов в день. Этот проект тратит ~500–700 в день на 10 каналов, так что запас огромный.

### Шаг 3. Создать репозиторий и загрузить файлы

1. На GitHub: **+** (справа вверху) → **New repository**.
2. Имя, например `youtube-top` → выбери **Public** (обязательно: у публичных бесплатные Actions и Pages) → **Create repository**.
3. Нажми **uploading an existing file** и перетащи туда ВСЕ файлы этой папки:
   - `collect.py`
   - `channels.txt`
   - `index.html`
   - `README.md`
   - папку `.github` целиком (внутри `workflows/collect.yml`)

   ⚠️ Если перетаскивание не подхватило папку `.github`, создай файл вручную: **Add file → Create new file**, в имени набери `.github/workflows/collect.yml` и вставь содержимое файла.
4. **Commit changes**.

### Шаг 4. Добавить API-ключ в секреты

1. В репозитории: **Settings** → **Secrets and variables** → **Actions**.
2. **New repository secret**.
3. Name: `YT_API_KEY` (ровно так). Secret: твой ключ из шага 2. → **Add secret**.

### Шаг 5. Указать свои каналы

Открой `channels.txt` в репозитории → карандаш (Edit) → впиши ссылки на каналы, по одной на строку:

```
https://www.youtube.com/@НазваниеКанала
https://www.youtube.com/channel/UCxxxxxxxxxxxxxxxxx
```

→ **Commit changes**. Каналы можно добавлять/убирать в любой момент так же.

### Шаг 6. Первый запуск сборщика

1. Вкладка **Actions** → если предложит, разреши workflows.
2. Слева выбери **Сбор статистики YouTube** → **Run workflow** → **Run workflow**.
3. Через 1–2 минуты запуск станет зелёным ✅. Если красный ❌ — открой его и посмотри лог (чаще всего опечатка в имени секрета или в ссылке на канал).

Дальше сборщик будет запускаться сам каждый час.

### Шаг 7. Включить страницу (GitHub Pages)

1. **Settings** → **Pages**.
2. Source: **Deploy from a branch** → Branch: `main`, папка `/ (root)` → **Save**.
3. Через пару минут страница будет доступна по адресу:
   `https://ТВОЙ_НИК.github.io/ИМЯ_РЕПОЗИТОРИЯ/`

Сохрани ссылку в закладки — открывается с любого устройства.

---

### Шаг 8 (по желанию). Управление каналами прямо на сайте

Каналы можно добавлять/удалять кнопкой **⚙ Каналы** на самой странице. Для этого странице нужен твой токен GitHub (вставляется один раз, хранится только в твоём браузере — посторонние менять список не смогут):

1. Открой [github.com/settings/personal-access-tokens/new](https://github.com/settings/personal-access-tokens/new).
2. Token name — любое. Expiration — можно поставить максимум (или No expiration, если доступно).
3. Repository access → **Only select repositories** → выбери свой репозиторий.
4. Permissions → Repository permissions → **Contents: Read and write** и **Actions: Read and write**.
5. **Generate token** → скопируй (показывается один раз).
6. На своей странице нажми **⚙ Каналы** → вставь токен → **Сохранить токен**.

После этого добавление канала на сайте сразу сохраняет его в репозиторий и запускает внеплановый сбор — новые данные появятся через ~2 минуты. Если токен истечёт, страница попросит вставить новый.

## Как пользоваться

- Открой страницу → видишь топ видео по приросту просмотров.
- Вкладки: **За 24 часа / За неделю / За месяц**.
- 10 видео на страницу, до 5 страниц (топ-50). Клик по карточке открывает видео.
- «Обновить» = перезагрузить страницу. Данные обновляются каждый час автоматически.
- **⚙ Каналы** — добавление и удаление каналов прямо на сайте (см. шаг 8). Альтернатива — редактировать `channels.txt` в репозитории.

## Важно знать

- **Первые данные** появятся после 2-го запуска сборщика (нужно минимум два снимка). Полноценный топ «за 24 часа» — через сутки, «за неделю» — через неделю, «за месяц» — через месяц. Пока история копится, видео помечаются значком ⏳ и прирост считается с начала наблюдения.
- **Активность**: GitHub отключает расписание, если в репозитории 60 дней нет активности. Сборщик сам коммитит данные каждый час, но на всякий случай заходи в репозиторий хотя бы раз в месяц-два.
- Отслеживаются **последние 50 видео** каждого канала (старые видео редко «выстреливают», но при желании лимит меняется в `collect.py` — `VIDEOS_PER_CHANNEL`).
- Точность окон: за 24ч — до часа, за неделю — до ~6 часов, за месяц — до суток (история прореживается для экономии места).
